# IT 资产管理系统 — 操作手册

## 一、系统概述

本系统用于 IT 资产的全生命周期管理，覆盖**资产登记 → 入库 → 领用/借用 → 归还 → 维修 → 报废**完整流程，配套库存预警、报表统计、用户管理等辅助功能。

**访问地址：** `http://localhost:5173`（开发模式）或 `http://localhost:8000`（生产模式）

**默认账号：** `admin` / `admin123`

---

## 二、登录系统

1. 打开浏览器访问系统地址，进入登录页
2. 输入用户名和密码
3. 点击「登录」按钮进入仪表盘

> 登录后 token 有效期 24 小时，超时自动跳转登录页。

---

## 三、仪表盘

登录后默认进入仪表盘，一览系统全局：

| 区域 | 内容 | 操作 |
|------|------|------|
| 顶部统计卡片 | 资产总数、在库、使用中、维修中、借出、已报废 | 点击无跳转，仅展示 |
| 资产状态分布图 | 柱状图，按 5 种状态统计数量 | 鼠标悬停查看具体数值 |
| 分类占比图 | 环形图，按分类统计资产占比 | 鼠标悬停查看百分比 |
| 最近操作 | 最近 8 条操作日志 | 滚动查看 |
| 库存预警 | 库存不足/过高的预警项 | 点击无跳转，仅提醒 |

## 四、资产管理

### 4.1 资产列表

**路径：** 侧边栏 → 资产管理 → 资产列表

**功能：** 查看、搜索、编辑、删除资产，生成二维码。

| 操作 | 步骤 |
|------|------|
| 查询资产 | 输入关键字（编号/名称/序列号），选择分类/状态，点击「查询」 |
| 重置筛选 | 点击「重置」清空条件并刷新 |
| 新增资产 | 点击「新增资产」跳转登记页（见 4.2） |
| 编辑资产 | 点击行内「编辑」按钮 |
| 删除资产 | 点击「删除」→ 确认弹窗 |
| 查看二维码 | 点击「二维码」→ 弹窗显示 PNG 图片，可右键保存 |

> 注意：已有出入库记录或维修记录的资产不可直接删除，需先清理关联数据。

### 4.2 资产登记

**路径：** 侧边栏 → 资产管理 → 资产登记，或从资产列表点击「新增资产」

**新增资产流程：**

1. 填写表单：资产名称（必填）、分类（必选）、品牌、型号、序列号
2. 填写采购信息：采购价格、采购日期、保修到期日
3. 选择归属：使用部门、仓库、供应商、存放位置
4. 填写备注（可选）
5. 点击「保存」

> 保存后系统自动生成资产编号，格式：`IT-YYYYMM-NNNN`（例：IT-202605-0001）

**编辑资产：** 在资产列表点击「编辑」，表单预填原有数据，修改后保存。

### 4.3 资产盘点

**路径：** 侧边栏 → 资产管理 → 资产盘点

**操作流程：**

1. 选择仓库和分类（可选），点击「筛选」列出待盘点资产
2. 逐项核对实物与系统信息
3. 勾选已核对的资产
4. 点击「批量标记已盘点」

## 五、库存管理

### 5.1 库存查询

**路径：** 侧边栏 → 库存管理 → 库存查询

**功能：** 查看所有库存状态，支持行内快捷操作。

| 筛选条件 | 说明 |
|----------|------|
| 关键字 | 模糊匹配资产编号/名称 |
| 分类 | 下拉选择资产分类 |
| 状态 | 在库/使用中/借出/维修中/已报废 |
| 仓库 | 下拉选择仓库 |

**行内快捷操作：**

| 按钮 | 出现条件 | 效果 |
|------|----------|------|
| 入库 | 资产不在库状态 | 弹出输入框填写说明，资产状态变为「在库」 |
| 出库 | 资产在库状态 | 弹出输入框填写领用说明，资产状态变为「使用中」 |
| 归还 | 资产为使用中/借出 | 资产状态变为「在库」 |
| 报废 | 任意状态 | 确认后资产状态变为「已报废」 |

### 5.2 入库管理

**路径：** 侧边栏 → 库存管理 → 入库管理

**操作流程：**

1. 在「资产」输入框输入关键字，远程搜索目标资产
2. 从下拉列表中选择资产
3. 选择入库类型：采购入库 / 借调入库
4. 选择目标仓库
5. 填写数量（默认 1）
6. 填写备注（可选）
7. 点击「确认入库」

> 入库后资产状态自动更新为「在库」。页面下方展示最近入库记录。

### 5.3 出库管理

**路径：** 侧边栏 → 库存管理 → 出库管理

**操作流程：**

1. 远程搜索资产（仅显示「在库」状态的资产）
2. 选择资产
3. 选择出库类型：领用出库 / 借用出库
   - **领用出库**：资产状态变为「使用中」，长期使用
   - **借用出库**：资产状态变为「借出」，需后续归还
4. 选择使用部门
5. 填写备注（可选）
6. 点击「确认出库」

> 出库后资产自动关联到所选部门。页面下方展示最近出库记录。

## 六、维保管理

### 6.1 维修记录

**路径：** 侧边栏 → 维保管理 → 维修记录

**新增维修：**

1. 点击「新增维修」
2. 远程搜索并选择报修资产
3. 填写故障描述、维修类型
4. 填写维修费用、报修日期
5. 点击「保存」

> 保存后资产状态自动更新为「维修中」。

**完成维修：**

- 在列表中点击「完成」按钮，资产状态自动恢复为「在库」

**编辑维修：** 点击「编辑」修改维修信息。

### 6.2 报废管理

**路径：** 侧边栏 → 维保管理 → 报废管理

**操作流程：**

1. 远程搜索并选择要报废的资产
2. 填写报废备注（可选）
3. 点击「确认报废」
4. 在确认弹窗中点击确定

> 报废操作不可撤销！资产状态变为「已报废」，同时在出入库记录中生成一条类型为「报废」的记录。页面下方展示报废历史列表。

---

## 七、报表统计

### 7.1 资产汇总

**路径：** 侧边栏 → 报表统计 → 资产汇总

展示 6 个统计卡片：资产总数、在库数、使用中数、维修中数、借出数、总价值。纯展示页面，无需操作。

### 7.2 库存统计

**路径：** 侧边栏 → 报表统计 → 库存统计

按三个维度展示统计表格：
- **按分类统计**：各类别资产数量和总价值
- **按状态统计**：各状态的资产数量
- **按部门统计**：各部门持有的资产数量

### 7.3 出入库报表

**路径：** 侧边栏 → 报表统计 → 出入库报表

**操作：**
1. 选择日期范围（开始/结束日期）
2. 选择出入库类型（可选）：采购入库、领用出库、借用出库、归还、报废
3. 点击「查询」

展示筛选结果：上方按类型汇总计数，下方明细列表。

## 八、系统管理

> 说明：以下模块均为基础数据的增删改查，操作模式统一。

### 通用操作模式

所有管理页面（部门/分类/供应商/仓库/预警）操作流程一致：

| 操作 | 步骤 |
|------|------|
| 新增 | 点击「新增xx」→ 填写表单 → 保存 |
| 编辑 | 点击行内「编辑」→ 修改 → 保存 |
| 删除 | 点击行内「删除」→ 确认弹窗 |

### 8.1 部门管理

管理使用部门，字段：名称、排序序号。

### 8.2 分类管理

管理资产分类，字段：名称、排序序号。默认预置：办公设备、网络设备、配件耗材、软件授权。

### 8.3 供应商管理

管理供应商信息，字段：名称、联系人、电话、地址、备注。

### 8.4 仓库管理

管理仓库信息，字段：名称、位置。默认预置：主仓库（A栋1层）。

### 8.5 库存预警

**功能：** 为指定仓库+分类设置库存阈值。

| 字段 | 说明 |
|------|------|
| 仓库 | 选择监控的仓库 |
| 分类 | 选择监控的分类 |
| 最低库存 | 低于此值触发「库存不足」告警 |
| 最高库存 | 高于此值触发「库存过高」告警 |

预警结果在**仪表盘**右下角展示。

### 8.6 操作日志

只读页面，记录所有用户的操作行为（谁、做了什么、什么时候）。支持分页浏览。

### 8.7 用户管理

> 仅管理员可操作。普通用户看不到此菜单。

| 操作 | 说明 |
|------|------|
| 新增用户 | 填写用户名、密码（至少6位）、姓名、角色 |
| 编辑用户 | 修改姓名、角色、状态（启用/禁用） |
| 重置密码 | 为指定用户设置新密码 |
| 删除用户 | 管理员不可删除，不可删除自己 |

**角色说明：**
- **管理员（admin）**：拥有全部权限
- **普通用户（user）**：可进行日常操作，但不能管理用户

## 九、完整业务流程示例

### 场景：新购一台笔记本电脑

```
1. [系统管理 → 分类管理]       确认「办公设备」分类存在
2. [系统管理 → 供应商管理]      录入供应商信息（如 Dell 中国）
3. [系统管理 → 仓库管理]       确认「主仓库」存在
4. [资产管理 → 资产登记]       录入笔记本信息 → 保存
   → 系统自动生成编号：IT-202605-0001
   → 状态自动设为「在库」
5. [资产管理 → 资产列表]       可查看/编辑/生成二维码
```

### 场景：员工领用笔记本

```
1. [库存管理 → 出库管理]       搜索「IT-202605-0001」
2. 选择资产 → 类型选「领用出库」
3. 选择使用部门 → 填写备注 → 确认出库
   → 资产状态变为「使用中」
   → 资产关联到所选部门
```

### 场景：笔记本故障送修

```
1. [维保管理 → 维修记录]       点击「新增维修」
2. 搜索资产 → 填写故障描述 → 保存
   → 资产状态变为「维修中」
3. 维修完成后 → 点击「完成」
   → 资产状态恢复为「在库」
```

### 场景：笔记本年限到期报废

```
1. [维保管理 → 报废管理]       搜索「IT-202605-0001」
2. 填写备注（如「已使用5年，性能不满足需求」）
3. 点击「确认报废」
   → 资产状态变为「已报废」
   → 出入库报表生成报废记录
```

## 十、Excel 批量导入

**适用场景：** 批量导入存量资产。

2. 列顺序：名称、分类ID、品牌、型号、序列号、采购价格、采购日期
3. 示例：

| 名称 | 分类ID | 品牌 | 型号 | 序列号 | 采购价格 | 采购日期 |

| iPhone 15 | 1 | Apple | A2846 | SN002 | 6999 | 2025-06-15 |

4. 调用导入接口（通过 API 或 Postman）：`POST /api/assets/import`，上传 Excel 文件

## 十一、常见问题

**Q：忘记管理员密码怎么办？**
A：删除 `backend/data/it_assets.db` 文件后重启后端，系统会重新初始化数据库（默认账号恢复为 admin/admin123）。**注意这会丢失所有数据！**

**Q：如何备份数据？**
A：复制 `backend/data/it_assets.db` 文件到安全位置即可。SQLite 单文件数据库，备份即复制。

**Q：JWT 密钥如何固定？**
A：设置环境变量 `IT_ASSET_SECRET=你的密钥`。未设置时每次重启随机生成，会导致之前登录的 token 失效。

**Q：如何修改端口？**
A：后端在 `main.py` 末尾修改 `port=8000`。前端在 `vite.config.js` 修改 `port: 5173`，并同步修改代理目标。

**Q：如何部署到服务器？**
A：构建前端 `npm run build`，将 `backend/` 和 `frontend/dist/` 目录部署到服务器，运行 `python main.py` 即可（后端自动挂载前端静态文件）。
